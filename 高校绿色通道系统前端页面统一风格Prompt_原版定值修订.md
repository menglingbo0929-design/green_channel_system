# 高校绿色通道系统前端页面统一风格 Prompt

## 一、使用说明

本 Prompt 用于生成“高校绿色通道系统”的一整套桌面端 Web 前端页面示意图。

后续每生成一个新页面时，只需要替换：

- `【页面名称】`
- `【当前登录角色】`
- `【页面具体功能】`

其他风格描述必须保持完全不变，以保证多个页面在布局、配色、字体、卡片、按钮、表格和导航栏方面具有统一的视觉风格。

每次必须将“统一基础 Prompt”和“当前页面功能描述”一起提交给 Image 2，禁止只写“保持上一张图片风格”。

## 二、统一基础 Prompt

```text
请生成一张“高校绿色通道系统”的桌面端 Web 前端页面高保真示意图。

【页面名称】
{{填写当前页面名称，例如：新生信息管理页、辅导员审核工作台、欠费确认页、统计分析页}}

【页面具体功能】
{{在这里填写该页面需要展示的功能、表格字段、按钮、筛选条件、统计数据或表单内容}}

一、整体设计风格

采用高校行政管理系统、学生资助管理平台和现代企业级后台管理系统的固定视觉风格，整体风格固定为正式、清晰、稳定、克制，体现高校信息化系统和学生资助业务的专业性。

页面采用蓝白色后台管理系统风格：

1. 页面左侧为深蓝色固定侧边导航栏；
2. 页面顶部为白色横向导航栏；
3. 主内容区域使用浅灰色背景；
4. 内容模块使用白色卡片承载；
5. 主色固定为科技蓝 #1677FF；
6. 成功状态固定使用绿色 #22C55E；
7. 警告状态固定使用橙色 #F59E0B；
8. 错误、退回或拒绝状态固定使用红色 #EF4444；
9. 次要信息固定使用中性灰色 #6B7280；
10. 禁止使用渐变、玻璃拟态、霓虹效果、夸张阴影和装饰性纹理。

二、固定页面结构

画面比例固定为16:9，分辨率固定为1600×900像素，生成一张完整的桌面端网页截图。

页面结构必须保持统一：

1. 左侧固定导航栏，宽度固定为224像素；
2. 顶部固定导航栏，高度固定为60像素；
3. 主内容区位于顶部导航栏下方；
4. 主内容区左右边距固定为24像素；
5. 主内容区有效内容宽度固定为1328像素；
6. 页面底部内容完整显示，不要截断主要操作按钮；
7. 所有区域边缘平整，不出现突出的台阶、缺口或不规则拼接。

三、左侧导航栏

左侧导航栏背景固定为深藏蓝色 #123B63，顶部固定显示一个28×28像素的白色线性盾牌图标，盾牌内部为白色毕业帽，以及系统名称：

“高校绿色通道系统”

导航菜单固定采用18×18像素、1.75像素描边的浅蓝白色线性图标和中文文字，普通图标及文字颜色固定为 #D9E6F2。

导航菜单固定使用以下项目，顺序不得改变：

首页
个人中心
新生信息管理
绿色通道申请
我的申请
审核管理
欠费确认
申请补录
统计报表
基础数据
政策与说明

当前选中的菜单背景固定为亮蓝色 #1677FF，文字和图标固定为 #FFFFFF。

菜单选中区域固定为4像素圆角矩形，禁止出现外凸、悬浮、缺口或不对齐现象。

左侧栏与顶部栏、主内容区的交界线必须平整，不能出现凸起、突出块或阶梯形边缘。

四、顶部导航栏

顶部导航栏背景固定为 #FFFFFF，底部固定使用1像素实线分隔线，颜色为 #E5E7EB。

左侧依次显示：

折叠菜单图标
首页
面包屑导航
当前页面名称

右侧显示：

消息通知图标
消息数量角标
圆形用户头像
用户名
当前角色
下拉箭头

顶部元素必须水平居中对齐，相邻元素间距固定为16像素。

五、主内容区域

主内容区域背景固定使用极浅灰色 #F4F6F9。

内容模块卡片背景固定为 #FFFFFF。

所有主要卡片圆角固定为2像素，整体视觉接近直角矩形。

不要使用明显的大圆角。

卡片之间间距固定为16像素。

卡片内部内边距固定为20像素。

卡片固定使用1像素实线浅灰色边框 #E5E7EB，不使用阴影。

禁止使用厚重阴影、悬浮玻璃效果和明显立体边框。

六、排版规范

所有界面文字使用简体中文。

字体固定采用现代无衬线中文字体：

Microsoft YaHei

页面标题固定采用20像素、字重600、行高28像素。

模块标题固定采用16像素、字重600、行高24像素。

正文、表格和表单文字固定采用14像素、字重400、行高22像素。

辅助说明固定采用12像素、字重400、行高20像素、颜色 #6B7280。

中文文字必须清晰、完整、自然，禁止出现乱码、错别字、异常字符或无意义英文。

标题、字段、数据和按钮之间需要形成清晰的信息层级。

七、按钮规范

主要操作按钮固定使用背景色 #1677FF、边框色 #1677FF、文字色 #FFFFFF 的实心矩形，例如：

新增
查询
保存
提交
下一步
审核通过
确认

次要按钮固定使用背景色 #FFFFFF、边框色 #1677FF、文字色 #1677FF，例如：

重置
返回
保存草稿
导出
打印

成功类按钮固定使用绿色 #22C55E。

危险类按钮固定使用红色 #EF4444。

警告或退回类按钮固定使用橙色 #F59E0B。

普通按钮高度固定为32像素，最小宽度固定为72像素，左右内边距固定为16像素，同组按钮间距固定为12像素，文字水平和垂直居中。

按钮圆角固定为4像素。

禁止使用胶囊形按钮和过大的圆角按钮。

八、表格规范

数据列表使用标准后台管理表格。

表格要求：

1. 表头背景固定为极浅蓝灰色 #F3F6FA；
2. 表头文字字重固定为600；
3. 表格边框固定使用1像素实线 #E5E7EB；
4. 表头和数据行高度均固定为44像素；
5. 数据列对齐整齐；
6. 操作列位于右侧；
7. 操作按钮固定使用高度28像素、最小宽度56像素的小型描边按钮；
8. 表格底部显示分页组件；
9. 状态标签高度固定为24像素、圆角固定为4像素；
10. 表格单元格左右内边距固定为12像素，文字禁止重叠，超长文字必须使用省略号。

九、表单规范

输入框、下拉框、日期选择器和单行文本框高度固定为32像素。

表单标签宽度固定为112像素，标签与输入控件间距固定为12像素，必须水平对齐。

必填项星号颜色固定为 #EF4444。

禁用字段背景固定为 #F3F4F6。

系统自动带出的学生基本信息固定采用背景色 #F8FAFC、边框色 #E5E7EB 的只读信息区域展示。

长表单固定使用分区标题划分；只有页面具体功能明确要求时才使用步骤条或标签页。

表单底部操作按钮区域固定右对齐，同组按钮间距固定为12像素。

十、图表规范

页面包含统计分析时，必须采用统一的 ECharts 后台图表风格。

图表类型只能从以下类型中选择；页面功能未明确指定时固定使用柱状图：

柱状图
折线图
饼图
环形图
数字统计卡片

图表颜色顺序固定为：主色 #1677FF、辅助色 #22C55E、#F59E0B、#60A5FA。

图表背景固定为 #FFFFFF。

坐标轴和图例文字固定为12像素、颜色 #6B7280；图表标题固定为16像素、字重600、颜色 #1F2937。

禁止使用三维图表、强渐变、复杂纹理和炫光效果。

十一、业务风格

系统属于高校学生资助和绿色通道业务平台，因此界面内容应当体现：

学生身份信息
学院、专业、年级和班级

学校审核
欠费确认
申请补录
统计报表

数据内容必须符合中国高校业务场景，禁止出现国外院校、国外角色名称或与学生资助无关的数据。

示例学院名称只能使用以下四个名称：

计算机科学与技术学院
经济管理学院
环境科学与工程学院
外国语学院

示例角色只能使用以下四个名称：

学生
辅导员
学院管理员
学校管理员

十二、视觉一致性要求

该页面属于同一套系统的系列页面，必须与其他页面保持一致：

1. 左侧导航栏宽度一致；
2. 顶部导航栏高度一致；
3. 系统名称和Logo位置一致；
4. 主内容区边距一致；
5. 卡片边框和直角风格一致；
6. 标题字号一致；
7. 表格样式一致；
8. 输入控件高度一致；
9. 按钮颜色、尺寸和圆角一致；
10. 图标使用统一的简洁线性风格；
11. 蓝色主色调一致；
12. 禁止改变页面框架；
13. 禁止增加插画、人物背景或装饰图案；
14. 不使用手机端或平板端布局；
15. 整体必须像同一个设计系统生成的页面。

十三、画面质量要求

生成完整、清晰、真实的高保真UI界面截图。

页面应具有实际可开发性，而不是概念海报。

所有组件必须按照8像素基础网格对齐；组件间距只能使用8、12、16、20或24像素。

画面正视，不使用透视角度，不展示电脑显示器、键盘或办公桌场景。

不要加入浏览器外框。

不要加入设计稿标注线。

不要加入水印。

不要加入与高校绿色通道无关的英文品牌。

不要出现悬浮在页面外的模块。

不要出现卡片边缘突起、左上角凸出、区域错位或不规则轮廓。

最终效果应当是一张专业、统一、简洁、正式、可用于项目原型展示的高校绿色通道管理系统前端页面。
```

## 四、示例：辅导员审核工作台

```text
本次需要生成的页面为：辅导员审核工作台。

当前登录角色：辅导员。

该页面主要包含：

1. 页面标题“绿色通道申请审核”；
2. 顶部显示待审核人数、已审核人数、退回人数和通过人数四个统计卡片；
3. 查询区域包含批次、学号、姓名、学院、专业、申请类型、申请状态和提交时间；
4. 设置“待审核”和“已审核”两个标签页；
5. 主表格字段包括申请编号、学号、姓名、学院、专业、申请类型、申请金额、礼包申请、提交时间、当前状态和操作；
6. 操作包括查看详情、审核通过、退回修改和审核不通过；
7. 表格支持多选和批量提交学院；
8. 页面右上角显示刷新和导出按钮；
9. 状态使用不同颜色的简洁标签；
10. 页面底部显示分页组件。

请严格沿用统一的蓝白色高校后台管理系统风格，保持深蓝色左侧导航栏、白色顶部栏、浅灰色背景、白色直角卡片和统一表格样式。
```

## 五、推荐使用方法

每次生成页面时，按照以下顺序组织内容：

```text
统一基础 Prompt
+
当前页面功能描述
+
必要的页面字段、按钮和示例数据
```

为了保持风格稳定，必须遵守：

1. 不修改统一基础 Prompt 中的配色、布局和组件规范；
2. 每张图都固定使用1600×900像素、16:9画面比例；
3. 每次明确当前登录角色；
4. 每次明确当前选中的左侧菜单；
5. 页面数据必须符合中国高校业务场景；
6. 不使用“自由发挥”“更有创意”等容易导致风格偏移的描述；
7. 新页面只调整主内容区域，不改变整体框架。
